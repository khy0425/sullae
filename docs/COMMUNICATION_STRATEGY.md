# 술래 커뮤니케이션 전략

## 핵심 철학

> **"채팅을 버렸기 때문에 게임이 살아난다"**

자유 채팅 없음 = 단점 ❌
자유 채팅 없음 = 의도적 설계 ⭕

---

## 왜 자유 채팅을 넣지 않는가?

### 잘못된 비교
| 당근마켓 | 술래 |
|---------|------|
| 자유 채팅 가능 → 장점? | 자유 채팅 없음 → 단점? |

### 정확한 비교
| 당근마켓 | 술래 |
|---------|------|
| **대화 중심** 모임 | **행동 중심** 게임 |

**이 둘은 경쟁하지 않는다. 역할이 다르다.**

---

## 자유 채팅이 필요한 상황 분석

채팅이 필요한 상황 3가지:
1. 장소 조율
2. 시간 변경
3. 규칙 협의

**술래의 해결책:**
| 상황 | 해결 방식 |
|------|----------|
| 장소 | 호스트가 확정 |
| 시간 | 호스트가 확정 |
| 규칙 | 게임 타입 + 커스텀 프리셋 |

→ **채팅이 필요한 이유를 구조적으로 제거**

---

## 자유 채팅의 숨은 비용

자유 채팅을 넣는 순간:
- 잡담 → 알림 폭탄
- 분쟁 → 중재 필요
- 욕설/성희롱 → 신고 시스템 필수
- **운영 비용 폭증**

술래는 이걸 **의도적으로 회피**한다.

---

## 대안: 통제된 소통 도구

### 1. 호스트 공지 (구현 완료)

```
"2번 출구 앞에서 만나요"
"경찰은 파란 모자 착용"
"도착하면 손 흔들어주세요"
```

→ **실제 필요한 소통의 80%**

### 2. 퀵 메시지 (정해진 문장만)

**자유 채팅 ❌ → 선택형 메시지 ⭕**

```
┌─────────────────────────────────────┐
│  퀵 메시지 선택                      │
├─────────────────────────────────────┤
│  📍 "지금 도착했어요"                │
│  ⏰ "5분 늦어요"                     │
│  ❓ "위치 변경됐나요?"               │
│  ✅ "게임 시작 가능해요"             │
│  🏃 "먼저 가서 기다릴게요"           │
│  👋 "오늘 못 갈 것 같아요"           │
└─────────────────────────────────────┘
```

**장점:**
- 욕설 불가
- 의미 없는 대화 불가
- 서버/운영 부담 최소
- 번역 용이 (다국어 지원)

**참고 UX:**
- Uber 드라이버-승객 메시지
- 배달 앱 "문 앞에 두세요"

### 3. 게임 중 채팅 완전 차단

**게임 중:**
- 공지 수신 ⭕
- 버튼 조작 ⭕
- 채팅 ❌

> 뛰어야 하는데 채팅하면, 그건 술래가 아니라 카톡 모임이다.

## 시나리오 테스트

### 퀵 메시지로 커버 가능한 상황

| 상황 | 퀵 메시지 |
|------|----------|
| 늦을 때 | "5분 늦어요" / "10분 늦어요" |
| 도착 알림 | "지금 도착했어요" |
| 위치 확인 | "위치 변경됐나요?" |
| 준비 완료 | "게임 시작 가능해요" |
| 불참 알림 | "오늘 못 갈 것 같아요" |
| 먼저 이동 | "먼저 가서 기다릴게요" |

### 커버 불가능한 상황 (호스트 공지 필요)

| 상황 | 대응 |
|------|------|
| 구체적 위치 설명 | 호스트 공지 업데이트 |
| 규칙 변경 | 호스트 공지 |
| 긴급 상황 | 호스트 공지 + 모임 취소 |

### 완전 커버 불가능한 상황 (앱 외부)

| 상황 | 대응 |
|------|------|
| 개인적 대화 | 카카오톡 등 외부 메신저 |
| 분쟁 해결 | 앱 외부에서 처리 (책임 회피) |

## 오프라인 진행 지원

### 철학

> **"다 모이면 직접 대화해도 된다. 앱은 보조 도구일 뿐."**

앱이 필수가 아니라, 게임을 더 쉽게 만드는 도구.
현장에서 직접 이야기해도 되지만, 앱을 쓰면 더 편하다.

### 1. 게임 가이드 배포

참가자들이 사전에 확인할 수 있는 게임 설명서:

```
┌─────────────────────────────────────┐
│  📖 경찰과 도둑 가이드               │
├─────────────────────────────────────┤
│  [기본 규칙]                        │
│  • 경찰: 도둑을 잡아 감옥에 가둠     │
│  • 도둑: 시간 내 잡히지 않으면 승리  │
│                                     │
│  [🏠 로컬 룰]                       │
│  • 감옥: 농구장 벤치               │
│  • 탈옥: 동료 터치로 가능           │
│  • 안전구역 없음                    │
│                                     │
│  [🎒 준비물]                        │
│  • 편한 운동화                      │
│  • 물                              │
│  • 팀 구분용 모자/밴드              │
│                                     │
│  [🛡️ 안전 수칙]                     │
│  • 차도 근처 금지                   │
│  • 위험한 곳에 숨지 않기            │
└─────────────────────────────────────┘
```

**장점:**
- 처음 참가하는 사람도 룰을 미리 파악
- 현장 설명 시간 단축
- 로컬 룰 사전 공유로 분쟁 방지

### 2. 진동 타이머

게임 중 화면을 보지 않아도 시간을 알 수 있음:

```
📳 진동 알림 패턴 (모든 게임 동일)

시작      →  짧게 2번     ━ ━
절반      →  길게 1번     ━━━
1분 전    →  짧게 1번     ━
종료      →  길게 2번     ━━━ ━━━
```

**핵심 원칙: 게임마다 다르게 하지 않는다**
- 경찰과 도둑 진동 vs 얼음땡 진동 → 혼란
- "진동 = 시간" 이라는 단일 개념
- 한 번 배우면 모든 게임에 적용

**장점:**
- 폰 꺼내지 않아도 됨
- 뛰면서도 시간 파악 가능
- 팀원 모두 동시에 알림
- 학습 비용 최소화

**사용 시나리오:**

```
철수: (뛰는 중) ...! (진동) "길게 1번? 절반 지났네!"
영희: (숨는 중) ...! (진동) "짧게 1번... 1분 남았어, 버텨!"
```

## 핵심 정리

| 항목 | 내용 |
|------|------|
| 자유 채팅 | 기본값이 아니다 |
| 술래 정체성 | 행동 최적화 앱 |
| 통제된 소통 | 약점이 아니라 설계 철학 |
| 결과 | 채팅을 버렸기 때문에 게임이 살아난다 |
